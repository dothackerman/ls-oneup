# Prototype Requirements

This folder converts the rough customer conversation notes into an implementable prototype scope.

Files:
- `01-customer-wishes.md`: Clarified customer needs and product goals.
- `02-milestones.md`: Shippable rapid-prototyping milestones.
- `03-open-questions.md`: Items to confirm with customer before or during implementation.

Working principles:
- Every milestone must be independently shippable.
- Each milestone must create usable customer value.
- Unknowns are tracked explicitly and validated early.
